Active Site Outputs

This file explains the active-site CSV outputs generated by rxn_analyzer.
The output filenames still use the historical site/reactive-site naming for compatibility.

Files
- <out_prefix>_site_states.csv
  Per-frame snapshot of each active site.
- <out_prefix>_site_events.csv
  Site-centered state-change events between adjacent frames.
- <out_prefix>_joint_site_reactions.csv
  Site-centered reactions using associated species as the reaction context.
- <out_prefix>_site_reaction_coupling.csv
  Unified table linking non-host transform events with active-site state changes.
- <out_prefix>_network_site_aware.graphml
  Graph containing species, reaction, and site_state nodes in one site-aware network.

site_states.csv columns
- frame
  Frame index used by the analyzer.
- site_id
  Stable user-defined active-site identifier.
- site_family
  User-defined site family/type, for example BAS or LAS_Ga.
- core_members
  Host atoms that anchor the site identity. These should remain stable across frames.
- intrinsic_members
  Non-host atoms that are part of the site by definition or initial setup.
- incorporated_members
  Previously external atoms/fragments that are now classified as part of the current site state.
- associated_members
  External atoms/fragments still treated as associated with the site, but not incorporated into the site state.
- state_members
  Union of intrinsic_members, incorporated_members, and associated_members.
- intrinsic_formula
  Element-count formula of intrinsic_members only.
- intrinsic_topology
  Topology-aware label of intrinsic_members when recognized; otherwise close to the formula.
- incorporated_formula
  Element-count formula of incorporated_members only.
- incorporated_topology
  Topology-aware label of incorporated_members when recognized.
- associated_formula
  Element-count formula of associated_members only.
- associated_topology
  Topology-aware label of associated_members when recognized.
- state_formula
  Element-count formula of the site state body, defined as intrinsic_members + incorporated_members.
- state_topology
  Topology-aware label of the site state body, defined as intrinsic_members + incorporated_members.
- state_label
  Human-readable summary label. If associated material exists, it is appended as + assoc(...).
- associated_species_labels
  Species labels for components that remain associated with the site and are not incorporated.
- attached_components
  Atom-index tuples for associated components linked to the site.
- descriptors
  JSON dictionary with extra counters and helper fields for debugging/analysis.

site_events.csv columns
- event_id
  Unique event identifier.
- frame
  Frame where the site change is recorded.
- site_id
  Active-site identifier.
- event_type
  Currently site_state_change.
- old_state
  Previous frame state_label.
- new_state
  Current frame state_label.
- added_members
  Atoms that entered state_members compared with the previous frame.
- removed_members
  Atoms that left state_members compared with the previous frame.
- associated_species_before
  Associated species labels in the previous frame.
- associated_species_after
  Associated species labels in the current frame.
- evidence_bonds
  Bond events touching the site neighborhood in this frame.

joint_site_reactions.csv columns
- event_id
  Unique event identifier.
- frame
  Frame where the joint reaction is recorded.
- site_id
  Active-site identifier.
- site_state_before
  Previous frame state_label.
- site_state_after
  Current frame state_label.
- reactants
  Previous associated species labels, used as the reaction context.
- products
  Current associated species labels, used as the reaction context.
- associated_species_before
  Same associated-species context as reactants, kept for readability.
- associated_species_after
  Same associated-species context as products, kept for readability.
- evidence_bonds
  Bond events touching the site neighborhood in this frame.

site_reaction_coupling.csv columns
- frame
  Frame index used by the analyzer.
- site_id
  Active-site identifier.
- site_family
  Active-site family/type.
- site_state_before
  Previous frame state_label for this site.
- site_state_after
  Current frame state_label for this site.
- site_changed
  Whether an active-site state-change event exists for this frame/site pair.
- has_transform
  Whether any non-host transform event was linked to this site on this frame.
- transform_event_ids
  Linked transform event ids.
- transform_types
  Linked transform event types.
- transform_reactants
  Reactant labels aggregated from linked transform events.
- transform_products
  Product labels aggregated from linked transform events.
- associated_species_before
  Associated species labels before the event/frame transition.
- associated_species_after
  Associated species labels after the event/frame transition.
- coupling_type
  One of reaction_only_on_site, reaction_with_site_change, or site_change_only.
- link_strength
  Link confidence: strong for atom-overlap evidence, medium for species-overlap evidence, none for site-only changes.
- evidence_bonds
  Supporting bond-event evidence collected from linked transforms or site events.

Reading tips
- core_members answer: which site is this?
- intrinsic_members + incorporated_members answer: what is the current site state?
- associated_members and associated_species_labels answer: what is still externally associated with the site?
- site_events.csv is the best file for site evolution.
- joint_site_reactions.csv is the best file for reaction-context interpretation.
- network_site_aware.graphml is the best graph view for seeing whether reactions happen on a site and whether the site state changes.
- In Gephi, try node labels from display_label, and inspect transform_equation, site_state_transition, coupling_type, and link_strength as custom attributes.
